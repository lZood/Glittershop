GUÍA PARA CREAR ENVÍO

 1. 1. Crea una cotización
 2. Envía una solicitud POST a /api/v1/quotations para obtener opciones de precio, servicio y carrier.
 3. 2. Consulta la cotización y escoge un “rate”
     * Usa GET en /api/v1/quotations/:id para ver los rates disponibles.
     * Elige y copia un rate (combinación de precio, servicio y carrier).

    Los rates son válidos 24 horas y la cotización se completa de forma progresiva (mira el campo is_completed).

 4. 3. Crea el envío
 5. Envía una solicitud POST a /api/v1/shipments con quotation_id y rate_id para generar el envío.
    Asegúrate de ingresar correctamente el nombre o código de la paquetería ("carrier_name"), así como los códigos de Carta Porte
    ("consignment_note"; "package_type") cuando sean requeridos. Consultar lista de códigos
    [https://help.skydropx.com/subcategorias-cda/api].


GUÍA PARA CREAR UN ENVÍO INTERNACIONAL

Los pasos para crear una guía internacional son muy similares, pero hay algunas diferencias que debes tener en cuenta. Si
necesitas más información sobre los valores que deben incluirse en cada parámetro, visita la documentación de cada endpoint.

 1. 1. Crea una cotización internacional
     * Envía una solicitud POST a /api/v1/quotations para obtener opciones de precio, servicio y carrier.
     * Las cotizaciones internacionales requieren el parámetro adicional "products" para calcular los aranceles de tu envío.
     * Asegúrate de elegir países válidos para tu cotización. Actualmente, solo soportamos envíos internacionales con origen
       México y los siguientes países como destino: Canadá, China, Colombia, España, EE. UU., Francia y Reino Unido.

 2. 2. Consulta la cotización y escoge un "rate"
     * Usa GET en /api/v1/quotations/:id para ver los rates disponibles.
     * Notarás que los "rates" de cotizaciones internacionales incluyen un valor adicional "import_duty_amount". Este valor
       representa los aranceles de dicha tarifa y no está incluido en el total de la misma.
     * Elige y copia un rate (combinación de precio, servicio y carrier).

 3. 3. Crea el envío
     * Envía una solicitud POST a /api/v1/shipments con el "rate_id" seleccionado para generar el envío.
     * Necesitarás agregar el parámetro adicional "customs_payment_payer", en el cual seleccionarás quién se encargará de pagar
       los aranceles del envío (tú o el destinatario). Si decides pagarlos tú, se descontarán de tus créditos junto con el total
       de la tarifa.
     * Adicionalmente, necesitas especificar el propósito del envío en el campo "shipment_purpose"
     * Por último, debes agregar nuevamente el parámetro "products" donde proporcionarás información más detallada sobre los
       artículos de tu envío.


GUÍA PARA CREAR UN ENVÍO CON MÚLTIPLES PAQUETES

Los pasos para crear un envío con múltiples paquetes son muy similares, pero hay algunas diferencias que debes tener en cuenta. Si
necesitas más información sobre los valores que deben incluirse en cada parámetro, visita la documentación de cada endpoint.

 1. 1. Crea una cotización multipaquete
     * Envía una solicitud POST a /api/v1/quotations para obtener opciones de precio, servicio y carrier.
     * Las cotizaciones de multipaquete requieren el parámetro adicional "parcels" donde debe estar incluido las dimensiones de
       cada paquete (Consultar la documentación de cada endpoint para más información de los parámetros requeridos).

 2. 2. Consulta la cotización y escoge un "rate"
     * Usa GET en /api/v1/quotations/:id para ver los rates disponibles.
     * Elige y copia el id de un rate.

 3. 3. Crea el envío
     * Envía una solicitud POST a /api/v1/shipments con el "rate_id" seleccionado para generar el envío.


GUÍA PARA IMPORTAR API A POSTMAN

Aqui encontraras los pasos necesarios para importar todas las llamadas al API directamente a una coleccion de Postman.

 1. 1. Copia la URL de la colección
     * Haz clic en el botón que se encuentra abajo para copiar la URL de la colección al portapapeles.
     * Esta URL contiene toda la especificación de nuestra API en formato OpenAPI compatible con Postman.

 2. 2. Abre Postman
     * Inicia la aplicación de Postman en tu computadora.
     * Si no tienes Postman instalado, puedes descargarlo desde https://www.postman.com/downloads/

 3. 3. Importa la colección
     * Haz clic en el botón "Import" en la parte superior izquierda de Postman.
     * Pega la URL que copiaste en el paso 1.
     * La importacion comenzará de forma automatica

 4. 4. ¡Listo para usar!
     * Ahora puedes explorar y probar todos los endpoints de nuestra API directamente desde Postman.
     * Cada endpoint incluye ejemplos y documentación para facilitar su uso.

/API/V1/OAUTH/TOKEN

POST

Resumen: Obtener un token de acceso

Descripción:

   POST /api/v1/oauth/token


El endpoint de token se utiliza para que el cliente obtenga un token de acceso presentando su concesión de autorización o token de
actualización.

Parámetros de la solicitud:

 * grant_type: El tipo de concesión utilizada, como refresh_token o client_credentials.
 * client_id: El identificador del cliente.
 * client_secret: La clave secreta del cliente.
 * refresh_token: El token de actualización (para el tipo de concesión refresh_token).
 * redirect_uri: La URI de redirección, debe ser la misma que la redirect_uri almacenada en la aplicación Cliente.
 * scope:Los scopes limitan el acceso de los tokens de acceso a partes específicas de la API. Al solicitar un token de acceso,
   puedes especificar uno o más scopes.

Ejemplo de solicitud:

   POST /api/v1/oauth/token
   Content-Type: application/x-www-form-urlencoded
grant_type=client_credentials&client_id=CLIENT_ID&client_secret=CLIENT_SECRET



EJEMPLO DE CUERPO DE SOLICITUD:

/API/V1/OAUTH/REVOKE

POST

Resumen: Revocar un token

Descripción:

   POST /api/v1/oauth/revoke


El endpoint de revocación permite a los clientes notificar al servidor de autorización que un token de actualización o acceso
previamente obtenido ya no es necesario.

Parámetros de la solicitud:

 * client_id: El identificador del cliente.
 * client_secret: La clave secreta del cliente.
 * token: El token que el cliente desea revocar.
 * token_type_hint: Una pista sobre el tipo de token enviado para revocación, por ejemplo, access_token o refresh_token.

Ejemplo de solicitud:

   POST /api/v1/oauth/revoke
   Content-Type: application/x-www-form-urlencoded

client_id=CLIENT_ID&client_secret=CLIENT_SECRET&token=TOKEN&token_type_hint=access_token

/API/V1/OAUTH/INTROSPECT

POST

Resumen: Obtener detalles del token

Descripción:

   POST /api/v1/oauth/introspect


El endpoint de introspección se utiliza para verificar el estado activo de un token de acceso o de actualización.

Parámetros de la solicitud:

 * client_id: El identificador del cliente.
 * client_secret: La clave secreta del cliente.

 * token_type_hint: Una pista sobre el tipo de token enviado para introspección, por ejemplo, access_token o refresh_token.

   POST /api/v1/oauth/introspect
   Content-Type: application/x-www-form-urlencoded

client_id=CLIENT_ID&client_secret=CLIENT_SECRET&token=TOKEN&token_type_hint=access_token

 * No incluir espacios.

Authorization: Bearer <token> (token proporcionado por nosotros)

Authorization: HMAC <firma>
Content-Type: application/json


La firma se genera utilizando el algoritmo HMAC con la función hash SHA-512, como se define en el RFC 6234
[https://www.rfc-editor.org/rfc/rfc6234.html], usando tu clave secreta. El HMAC se calcula sobre el cuerpo crudo de la solicitud
(bytes exactos, sin formato) y se codifica como una cadena hexadecimal en minúsculas.

Cuerpo de la solicitud El cuerpo de la solicitud contiene información relevante sobre el evento notificado.

Métodos de autenticación

1. Autenticación con Bearer Token

Utiliza el token estático proporcionado para una autenticación sencilla (menos segura).

2. Autenticación HMAC (Recomendada)

 * Crear la cadena de firma: raw_request_body.
 * Calcular HMAC-SHA512 utilizando tu clave secreta.
 * Enviar la firma en el encabezado Authorization.

Información adicional Por razones de privacidad, no podemos compartir toda la información directamente en el webhook. Sin embargo,
los clientes pueden hacer una solicitud adicional a la API usando los enlaces proporcionados en el cuerpo de la respuesta para
obtener los detalles completos del evento. Para la documentación completa de la API:
 * Envíos: Documentación de envíos [/es-MX/api-docs#get-api-v1-shipments-id]
 * Órdenes: Documentación de órdenes [/es-MX/api-docs#get-api-v1-orders-id]

Seguridad Asegúrate de que tu URL de webhook use HTTPS.

1. Para autenticación con Bearer Token:

 * Trata el token como credenciales sensibles.
 * Rota los tokens periódicamente.
 * Valida todas las solicitudes entrantes antes de procesarlas.

2. Para autenticación HMAC:

 * Almacena tu clave secreta de forma segura.
